B.

내가 사용한 기술과 선정 이유

로봇의 인식·위치·대화를 각각 담당하는 모듈을 ROS 기반으로 연결했습니다.

YOLOv8

로봇 카메라 기반 객체탐지. 데이터셋·하이퍼파라미터를 바꿔가며 v1~v7까지 반복 학습·성능 검증.

RTAB-Map (VSLAM)

시각 기반 SLAM으로 로봇 위치/맵을 추정. 추정 좌표를 외부 모듈로 실시간 전송.

ROS 2 (gae_ws)

비전·SLAM·음성 모듈 간 토픽 기반 통신. 워크스페이스 구조와 실행 스크립트 구성.

MQTT

로봇 ↔ 외부 시스템 간 경량 메시지 전송. 좌표/상태 브릿지에 사용.

OpenAI API

AI 음성비서 '싸라'의 자연어 응답. 음성 입력을 의도로 해석해 로봇 동작/안내로 연결.

C.

담당 기능

로봇의 인식(YOLO) · 위치(VSLAM) · 대화(음성비서) 세 축을 담당했습니다.

YOLOv8 객체탐지 v1 → v7 고도화

in the end

모델은 한 번에 완성되지 않는다

YOLO를 v1에서 v7까지 반복하며, AI 모델은 한 번에 완성되는 게 아니라 데이터와 설정을 바꿔가며 성능을 검증하고 기록으로 남기는 사이클이라는 것을 체감했습니다. 로봇의 인식·위치·대화를 따로 만든 뒤 좌표로 잇는 경험은, 이후 특화 프로젝트의 시뮬↔웹 통합 파이프라인으로 자연스럽게 이어졌습니다.
